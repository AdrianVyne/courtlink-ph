# Security Policy

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities. Until a dedicated security address is configured, use GitHub's private vulnerability reporting feature on the repository.

Include affected components, reproduction steps using synthetic data, expected impact, and any proposed mitigation. Do not access, retain, or disclose other users' data while investigating.

## Supported versions

Only the latest commit on `main` is supported before the first stable release.
